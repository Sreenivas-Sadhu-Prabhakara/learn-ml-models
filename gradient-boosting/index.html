Lesson 06 · Classification + regression

A team that corrects itself.

Gradient Boosting builds small trees in sequence. Each round concentrates on what the current team still handles badly. You will tune that sequence on training folds, open the test set once, and finish with a leaderboard that belongs to your problem—not this course.

THE IDEA

Many small corrections, added together

A Random Forest grows many varied trees independently and averages their answers. Gradient Boosting is a relay: tree 1 starts the prediction, tree 2 learns a small correction, tree 3 corrects what remains, and the process continues.

Number of rounds

n_estimators controls how many correction trees join the sequence. Too few can underfit; too many can waste time or overfit.

Size of each correction

learning_rate controls how strongly each new tree changes the team. Smaller steps usually need more rounds.

The model works for classification and regression. It does not require numeric scaling, but the shared preprocessor still fills missing values and encodes categories.

STEP 2 OF 6

Choose the dials with training-only cross-validation

This search compares combinations across folds made only from X_train. The held-out test rows remain closed. Start with this modest grid; a large dataset may take several minutes.

CELL 2 · CROSS-VALIDATED SEARCH
from sklearn.model_selection import GridSearchCV

parameter_grid = {
    "model__n_estimators": [50, 100, 200],
    "model__learning_rate": [0.03, 0.1, 0.3],
    "model__max_depth": [2, 3],
}

gradient_search = GridSearchCV(
    estimator=gradient_pipeline,
    param_grid=parameter_grid,
    scoring=PRIMARY_SCORING,
    cv=make_cv(),
    n_jobs=-1,
    refit=True,
)
gradient_search.fit(X_train, y_train, **CV_FIT_PARAMS)

print("Best training-CV settings:")
for name, value in gradient_search.best_params_.items():
    print(f"  {name}: {value}")

if TASK == "classification":
    print(f"Best mean CV balanced accuracy: {gradient_search.best_score_:.3f}")
else:
    print(f"Best mean CV MAE: {-gradient_search.best_score_:.3f}")
CHECKPOINT

Write down which pair won. A larger number of rounds is not automatically better, and a higher learning rate is not automatically faster learning in the useful sense.

If this search is too slow

Use [50, 100] for the rounds, [0.05, 0.1] for the rate, and [2] for depth. Compute limits are part of model selection; reducing the search is better than abandoning honest validation.

STEP 5 OF 6

Measure which original columns matter

Permutation importance shuffles one original feature at a time and measures how much the final score deteriorates. Because it wraps the whole pipeline, categorical features stay grouped under their original column names.

CELL 5 · PERMUTATION IMPORTANCE
from sklearn.inspection import permutation_importance

gradient_importance = permutation_importance(
    best_gradient,
    X_test,
    y_test,
    scoring=PRIMARY_SCORING,
    n_repeats=5,
    random_state=RANDOM_STATE,
    n_jobs=-1,
)

importance_table = pd.DataFrame({
    "feature": X_test.columns,
    "importance": gradient_importance.importances_mean,
    "variation": gradient_importance.importances_std,
}).sort_values("importance", ascending=False)

display(importance_table)

importance_table.sort_values("importance").plot.barh(
    x="feature",
    y="importance",
    xerr="variation",
    legend=False,
    figsize=(8, max(4, len(importance_table) * 0.35)),
)
plt.xlabel("Score lost after shuffling this feature")
plt.title("Gradient Boosting permutation importance")
plt.tight_layout()
plt.show()

Importance is not causation

A feature can be useful because it is a proxy, correlated with another feature, or produced by the same process as the target. Correlated columns may also share importance. Treat this as a description of the fitted model, not a law about the world.

OPTIONAL DEEP DIVE

Estimated probabilities for classification

For a classification project, inspect the model's estimated class probabilities. These numbers can rank confidence, but they are not guaranteed to be calibrated and should never be described as certainty.

OPTIONAL CELL · PROBABILITY PREVIEW
if TASK != "classification":
    print("Probability output applies to classification only.")
else:
    class_names = best_gradient.named_steps["model"].classes_
    probabilities = best_gradient.predict_proba(X_test)

    probability_preview = X_test.head(10).copy()
    probability_preview["truth"] = y_test.head(10).values
    probability_preview["prediction"] = gradient_predictions[:10]
    probability_preview["largest_estimated_probability"] = (
        probabilities[:10].max(axis=1)
    )

    print("Probability column order:", list(class_names))
    display(probability_preview)
Why “estimated probability” instead of “confidence”?

A model can output 0.90 and still be correct much less than 90% of the time. Calibration must be checked separately on validation data. Probability describes the model's fitted scale, not how trustworthy the dataset or world is.

COURSE RECAP

The reusable workflow was the real model

  1. Define one row, the target, available features, and the costly mistake.
  2. Choose a random, time, or group split that resembles deployment.
  3. Learn preprocessing from training data inside a pipeline.
  4. Build a dummy baseline before using a sophisticated estimator.
  5. Choose dials with cross-validation—not the final test set.
  6. Open the test once and inspect errors, not only the headline metric.
  7. Prefer the model whose trade-offs fit the actual decision.
