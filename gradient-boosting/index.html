Lesson 06 · Classification

Gradient Boosting: the team that learns from its mistakes

The final lesson. A Random Forest asks 200 trees to answer independently and vote. Gradient Boosting builds its trees one at a time — each new tree trained specifically on the deals the previous ones got wrong. Then, the finale: every model from this course sits the exact same exam, and we print the course leaderboard.

⏱ about 30 minutes 🧠 builds on every classification lesson 🏆 ends with the course leaderboard
Start the lesson ↓

The whole idea, in one picture

Think of a study group preparing for one big exam. Week 1, the group attempts the full question paper. Week 2, they don't re-study everything — they spend the whole week only on the questions they got wrong. Week 3 targets whatever is still wrong. Every new week's work is a specialist in last week's mistakes.

That is Gradient Boosting. It builds small decision trees one after another: tree 2 is trained to correct tree 1's errors, tree 3 to correct what's still wrong, and so on for 100 rounds. The final answer adds up all the corrections. The word gradient is just the maths compass that points each new tree toward "fewer mistakes" — you never have to touch the maths yourself.

A forest polls everyone at once. Boosting builds tree 2 to fix tree 1's mistakes, tree 3 to fix what's left — 100 rounds of corrections, added together.

The task is the same exam as the whole course: look at three numbers — quantity, price, value in crore — and guess whether a deal is BULK (big trading spread across a normal day) or BLOCK (one giant pre-negotiated trade in a special window). Same data, same 50/50 balancing, same 75/25 split — so today's score can be compared, fairly, with every lesson before it.

Choose the clues, then split 75/25

Three clues in, one word out — and a quarter of the deals locked away as the final exam. Notice what's missing: no StandardScaler today. STEP 4 OF 7

Train the team, take the exam

Out of the box, the team runs 100 rounds of corrections, and each round's fix is applied gently (more on that dial in Step 5). Training takes a few seconds — the trees are built one after another, so this is genuinely 100 models being trained in sequence.

STEP 5 OF 7

The two dials, in plain words

Gradient Boosting has two dials that matter most:

  • n_estimators — the number of correction rounds. How many weeks the study group meets. Default: 100.
  • learning_rate — how big each correction is. At the default 0.1, each new tree's fix is applied at only 10% strength — like salting dal a pinch at a time and tasting, instead of tipping in a spoonful. Small and careful needs more rounds; big and bold gets there fast but can overshoot.

Let's turn each dial while holding the other at its default. This cell trains ten separate teams, so give it a minute or two.

CELL 5 · PYTHON
# Step 5: The two dials — correction rounds, and the size of each correction
print("Dial 1: number of correction rounds (learning_rate stays at 0.1)")
rounds_list = [25, 50, 100, 200, 400]
round_scores = []
for rounds in rounds_list:
    model = GradientBoostingClassifier(n_estimators=rounds, random_state=42)
    model.fit(X_train, y_train)
    score = accuracy_score(y_test, model.predict(X_test))
    round_scores.append(score)
    print(f"  {rounds:>3} rounds  →  accuracy {score:.1%}")

print()
print("Dial 2: size of each correction (rounds stay at 100)")
for rate in [0.01, 0.05, 0.1, 0.3, 1.0]:
    model = GradientBoostingClassifier(learning_rate=rate, random_state=42)
    model.fit(X_train, y_train)
    score = accuracy_score(y_test, model.predict(X_test))
    print(f"  learning_rate {rate:<4}  →  accuracy {score:.1%}")

plt.plot(rounds_list, round_scores, marker="o")
plt.xlabel("Number of correction rounds (n_estimators)")
plt.ylabel("Accuracy on the test set")
plt.title("More rounds of corrections")
plt.grid(True, alpha=0.3)
plt.show()

Reading the dials: more rounds kept helping here — 400 rounds reached 82.7%. Bigger corrections also scored well on this dataset, but be careful copying that habit: on messier data, learning_rate=1.0 is notorious for overshooting — like turning the shower tap a full rotation each time. The cautious default exists for a reason, and the two dials trade off: tiny corrections need many more rounds (the homework proves this on real numbers).

One honest confession: to keep this lesson short, we compared dial settings using the test set. In a real project that's cheating — you'd pick them using a separate slice of the training data (called a validation set), and touch the test set exactly once, at the very end.

# Step 7: THE FINALE — every model from the course, same exam, one leaderboard
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import RandomForestClassifier

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)   # KNN and Logistic Regression need scaled clues
X_test_scaled  = scaler.transform(X_test)

contestants = {
    "KNN (k = 5, scaled clues)":          (KNeighborsClassifier(n_neighbors=5), True),
    "Logistic Regression (scaled clues)": (LogisticRegression(max_iter=1000, random_state=42), True),
    "Decision Tree (depth 3)":            (DecisionTreeClassifier(max_depth=3, random_state=42), False),
    "Decision Tree (unlimited depth)":    (DecisionTreeClassifier(random_state=42), False),
    "Random Forest (200 trees)":          (RandomForestClassifier(n_estimators=200, random_state=42), False),
    "Gradient Boosting (today's model)":  (GradientBoostingClassifier(random_state=42), False),
}

leaderboard = [("Coin flip (the baseline)", 0.50)]
for name, (model, needs_scaling) in contestants.items():
    clues_train = X_train_scaled if needs_scaling else X_train
    clues_test  = X_test_scaled  if needs_scaling else X_test
    model.fit(clues_train, y_train)
    leaderboard.append((name, accuracy_score(y_test, model.predict(clues_test))))

leaderboard.sort(key=lambda row: row[1], reverse=True)

print("COURSE LEADERBOARD — same 4,469 hidden test deals for every model")
print("=" * 60)
for rank, (name, acc) in enumerate(leaderboard, start=1):
    print(f"{rank}. {name:<36} {acc:.1%}")

Why this family still matters most. Out in the real world, on tables of data like this one — loans, insurance claims, customer churn, crop yields — boosted trees win more machine-learning competitions than any other method, including fancy deep learning. The winning entries almost always use gradient boosting's turbo-charged cousins, XGBoost and LightGBM: the same one-tree-fixes-the-last idea you just built, engineered for speed and tuned hard (on a validation set, of course). With this lesson, you haven't just finished a course — you've met the practical state of the art for everyday tabular data.

How to judge any model — the cheat sheet

One last time — the four questions you'll reuse on every ML project you ever do:

QuestionToolWhat to look for
1. Better than dumb luck?Compare with a baselineOur 80.4% vs a 50% coin flip. If a model can't clearly beat the dumbest strategy, it learned nothing.
2. Where does it fail?Confusion matrixBig numbers on the diagonal. Ours misses only 231 of 2,235 BLOCKs but mislabels 643 BULKs — know which mistake your model makes more.
3. Can I trust a "yes"? Does it miss?Precision & recallPrecision = trustworthiness of its claims (0.76 for BLOCK). Recall = how much it misses (0.90 — barely misses). Pick the one your problem cares about.
4. Was the exam fair?Train/test splitScore must come from data the model never saw — our 4,469 locked-away deals. And the leaderboard is only fair because every model sat the same exam.

Rough accuracy guide for a balanced 50/50 problem like ours: ~50% = learned nothing · 60–75% = learned something · 75–90% = solid · 95%+ = be suspicious — usually the answer leaked into the clues. Too perfect is not a triumph; it's a symptom.

The course, in one look back

Five classification lessons, and the recipe never once changed: load → look → balance → split → train → judge against a baseline. Only the model in the middle was swapped out. Along the way you met the whole family:

  1. KNN — judge a deal by its most similar neighbours, and let them vote (Lesson 01, 79.8%).
  2. Logistic Regression — one weighted formula that adds up the evidence (75.3%).
  3. Decision Trees — a game of 20 questions; three questions keep it honest (78.7%), unlimited questions memorise (83.0%).
  4. Random Forest — hundreds of trees, each seeing a different slice, majority wins (83.8%).
  5. Gradient Boosting — trees in sequence, each new one trained on the last one's mistakes (80.4% out of the box, 82.7% with more rounds).

And the habits that matter more than any model: never trust a score without a baseline, always look at the confusion matrix, keep the test set locked away, tune dials on a validation set, and treat a too-perfect score as a symptom, not a triumph.
