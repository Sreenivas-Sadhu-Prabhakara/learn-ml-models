IDEA 1

One neighbourhood, two kinds of answer

Imagine placing every row on a map whose directions are its feature values. KNN locates a new row, measures its distance from the training rows, and consults the nearest K.

Classification

The neighbours vote for their class labels. The label with the strongest vote becomes the prediction.

Regression

The neighbours contribute their numeric targets. Their average becomes the prediction.

KNN does not learn a compact formula. Its memory is the training data itself, so prediction gets slower as the dataset grows.

“Near” depends on units

If one numeric feature ranges from 0 to 1 and another from 0 to 1,000,000, the larger scale can dominate the distance. This lesson standardises numeric features inside the pipeline. The scaler learns from each training fold only.

DECISION

What does K control?

Choice Likely behaviour Question to ask
Small K Flexible and sensitive to individual rows Is the model following noise or unusual duplicates?
Large K Smoother and pulled toward the common answer Is a real local pattern being averaged away?

There is no universally correct K. Your training folds can compare candidates without revealing the final exam.

Deep dive: why distance can struggle in many dimensions

As the number of encoded features grows, rows tend to look similarly far apart. This is often called the curse of dimensionality. IDs, very high-cardinality categories, and hundreds of weak columns can make a nearest-neighbour model both slow and unhelpful. Better feature judgment usually matters more than a wider K search.

CELL 3 · CROSS-VALIDATION SEARCH 
cv = make_cv()

# Find the smallest training fold, without touching X_test.
if "groups" in CV_FIT_PARAMS:
    cv_splits = cv.split(
        X_train,
        y_train,
        groups=CV_FIT_PARAMS["groups"],
    )
else:
    cv_splits = cv.split(X_train, y_train)

fold_train_sizes = [
    len(fold_train_index)
    for fold_train_index, _ in cv_splits
]
largest_safe_k = min(31, min(fold_train_sizes))

candidate_k = [
    k for k in [1, 3, 5, 7, 9, 15, 21, 31]
    if k <= largest_safe_k
]

if not candidate_k:
    raise ValueError("There are too few training rows to fit KNN.")

knn_search = GridSearchCV(
    estimator=knn_pipeline,
    param_grid={"model__n_neighbors": candidate_k},
    scoring=PRIMARY_SCORING,
    cv=cv,
    n_jobs=-1,
    return_train_score=True,
    error_score="raise",
)
knn_search.fit(X_train, y_train, **CV_FIT_PARAMS)

raw_cv = pd.DataFrame(knn_search.cv_results_)

if TASK == "classification":
    cv_summary = pd.DataFrame({
        "k": raw_cv["param_model__n_neighbors"].astype(int),
        "mean_validation_balanced_accuracy": raw_cv["mean_test_score"],
        "fold_to_fold_variation": raw_cv["std_test_score"],
        "rank": raw_cv["rank_test_score"],
    })
else:
    cv_summary = pd.DataFrame({
        "k": raw_cv["param_model__n_neighbors"].astype(int),
        "mean_validation_MAE": -raw_cv["mean_test_score"],
        "fold_to_fold_variation": raw_cv["std_test_score"],
        "rank": raw_cv["rank_test_score"],
    })

display(cv_summary.sort_values("k").reset_index(drop=True))
print("Selected K:", knn_search.best_params_["model__n_neighbors"])

CELL 5 · ERROR REVIEW 
outcomes = pd.DataFrame(
    {
        "__actual__": np.asarray(y_test),
        "__predicted__": np.asarray(knn_predictions),
    },
    index=X_test.index,
)
review = pd.concat([outcomes, X_test], axis=1)

if TASK == "classification":
    mistakes = review[
        review["__actual__"] != review["__predicted__"]
    ]
    print(f"Mistakes: {len(mistakes):,} of {len(review):,}")

    if mistakes.empty:
        print("No test mistakes. Audit for leakage before celebrating.")
    else:
        mistake_types = (
            mistakes
            .groupby(["__actual__", "__predicted__"], dropna=False)
            .size()
            .sort_values(ascending=False)
            .rename("rows")
        )
        display(mistake_types.to_frame())
        display(mistakes.head(10))

else:
    review["__residual__"] = (
        review["__actual__"] - review["__predicted__"]
    )
    review["__absolute_error__"] = review["__residual__"].abs()

    display(
        review
        .sort_values("__absolute_error__", ascending=False)
        .head(10)
    )

    plt.figure(figsize=(7, 4))
    plt.scatter(
        review["__predicted__"],
        review["__residual__"],
        alpha=0.55,
    )
    plt.axhline(0, color="black", linestyle="--")
    plt.xlabel("Predicted value")
    plt.ylabel("Residual = actual - predicted")
    plt.title("KNN residual check")
    plt.show()
CHECKPOINT

Classification: identify the most common direction of error and inspect several rows. Regression: look for curves, funnels, clusters, and extreme residuals rather than expecting a perfect cloud.

RECAP

What you can now explain

  1. KNN predicts from nearby training rows: a vote for classification, an average for regression.
  2. Distance-based models need comparable numeric scales.
  3. A pipeline keeps imputation, encoding, and scaling inside each training fold.
  4. K is selected with cross-validation on training data, never by chasing test performance.
  5. A metric starts the investigation; error rows reveal whether the model is useful.
