The whole idea, in one picture

Imagine you move to a new city and want to know whether a house is in a posh area or a normal area. The obvious trick: look at the five nearest houses and go with the majority.

That is the entire algorithm. To classify something new, KNN finds the K most similar examples it has already seen, lets them vote, and the majority wins. "Similar" just means close in numbers — two deals with almost the same quantity, price and value are neighbours.

The mystery deal's 5 nearest neighbours: 3 BLOCK, 2 BULK. The vote says BLOCK — 3 to 2.

What you're going to predict

Every day, unusually large share trades on the Indian exchanges (almost all NSE, a handful from BSE) are published in two categories. Your model will look at just three numbers — how many shares, at what price, worth how many crores — and guess the category.

BULK DEAL

Someone buys or sells more than 0.5% of a company's shares over a normal trading day. Big, but spread across the day's trading.

BLOCK DEAL

One single, giant, pre-negotiated trade — crores of rupees in one shot — executed in a special trading window. These tend to be much bigger in value.

"Block deals tend to be bigger" is the pattern we hope the model discovers on its own. We never tell it that rule — it has to learn it from the ~13,400 training deals we'll carve out in Step 6.

STEP 4 OF 12

Make it a fair 50/50 contest

The simplest fix: keep all 8,938 BLOCK deals, and randomly pick an equal number of BULK deals. Now blind guessing only scores 50% — the model has to genuinely learn something to beat that.

CELL 4 · PYTHON
# Step 4: Balance the dataset — equal number of BULK and BLOCK deals
block_deals = deals[deals["deal_type"] == "BLOCK"]
bulk_deals  = deals[deals["deal_type"] == "BULK"].sample(n=len(block_deals), random_state=42)

data = pd.concat([block_deals, bulk_deals])
data["deal_type"].value_counts()

deal_type

BULK     8938
Name: count, dtype: int64

What's random_state=42? It means "pick randomly, but pick the same random deals every time" — so everyone in the room gets identical results. The number 42 is a programmer in-joke; any fixed number works.

STEP 6 OF 12

Split the data: 75% training, 25% testing

This is the most important honesty rule in all of machine learning:

  • 75% training set — the model studies these deals, answers included.
  • 25% test set — locked away. The model never sees these while learning. This is the final exam.

Testing a model on deals it already studied is like grading a student on questions they memorised the night before — the mark means nothing.

CELL 6 · PYTHON
# Step 6: Split — 75% for learning, 25% hidden away for the exam
X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    train_size=0.75,     # 75% training, 25% testing
    random_state=42,     # same split for everyone
    stratify=y,          # keep the 50/50 mix in both halves
)

print("Deals for training:", len(X_train))
print("Deals for testing: ", len(X_test))
WHAT YOU SHOULD SEE
Deals for training: 13407
Deals for testing:  4469
STEP 7 OF 12

Put all clues on the same scale

KNN works by measuring distance between deals. But look at the first row you just saw in Step 5: quantity is 4,59,83,509, the price is ₹400, and the value is ₹1,839 crore — three numbers on wildly different scales. Without fixing this, the giant quantity numbers bully the distance calculation and the other clues barely count.

StandardScaler converts every column so they all speak at the same volume. One subtle detail: we learn the scaling from the training data only (fit_transform), then apply it to the test data (transform) — the exam paper must never influence the study notes.

CELL 7 · PYTHON
# Step 7: Scale the features so no single clue dominates
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)   # learn the scale from training data
X_test_scaled  = scaler.transform(X_test)        # apply that same scale to test data

print("All clues are now on the same scale.")
WHAT YOU SHOULD SEE
All clues are now on the same scale.

Feels like nothing happened? Step 10 has a surprise that proves this line was worth several percentage points of accuracy.

STEP 8 OF 12

Train the model

n_neighbors=5 is the K in KNN: to judge a new deal, look at its 5 most similar deals and take a vote. Training is instant — KNN simply memorises the training deals.

CELL 8 · PYTHON
# Step 8: Train the KNN model (K = 5 neighbours)
knn = KNeighborsClassifier(n_neighbors=5)
knn.fit(X_train_scaled, y_train)

print("Model trained on", len(X_train), "deals.")
WHAT YOU SHOULD SEE
Model trained on 13407 deals.
STEP 9 OF 12

Exam time!

The model now predicts the deal type for all 4,469 test deals — the 25% it has never seen — and we check its answers against the truth.

CELL 9 · PYTHON
# Step 9: Predict the hidden 25% and check the score
predictions = knn.predict(X_test_scaled)

accuracy = accuracy_score(y_test, predictions)
print(f"Accuracy: {accuracy:.1%}")
WHAT YOU SHOULD SEE
Accuracy: 79.8%

Is 79.8% good? Always compare against the dumbest possible strategy. A coin toss scores ~50% here. Our model gets 4 out of 5 deals right using only three numbers per deal — it genuinely discovered that block deals tend to be bigger-value trades. Nobody told it that.

TermPlain-language meaning
PrecisionWhen the model says BLOCK, how often is it actually BLOCK? (How trustworthy are its claims?)

STEP 11 OF 12

Tune K: how many neighbours should vote?

K is a dial we chose freely (we picked 5). Small K = jumpy decisions based on very few neighbours. Huge K = the vote averages over so many neighbours it blurs the pattern. Let's try a range.

CELL 13 · PYTHON
# Step 11: Try different values of K and plot the results
k_values = [1, 3, 5, 7, 9, 11, 15, 21, 31, 51]
scores = []

for k in k_values:
    model = KNeighborsClassifier(n_neighbors=k)
    model.fit(X_train_scaled, y_train)
    score = accuracy_score(y_test, model.predict(X_test_scaled))
    scores.append(score)
    print(f"K = {k:>2}  →  accuracy {score:.1%}")

plt.plot(k_values, scores, marker="o")
plt.xlabel("K (number of neighbours who vote)")
plt.ylabel("Accuracy on the test set")
plt.title("Choosing K")
plt.grid(True, alpha=0.3)
plt.show()
WHAT YOU SHOULD SEE

…plus a line chart of the same numbers.

Careful with K = 1! It scores highest here, but with only one neighbour voting, a single odd or duplicated deal decides the answer — the model equivalent of asking exactly one person for directions. (Our data even contains near-twin rows: the buy side and sell side of the same trade, which flatters K = 1.) Odd, modest values like 5–15 are the safe standard choice; odd numbers also mean the vote can never tie.

One honest confession: to keep this lesson short, we compared K values using the test set. In a real project that's cheating — you'd pick K using a separate slice of the training data (called a validation set), and touch the test set exactly once, at the very end.

How to judge any model — the cheat sheet

You'll reuse these four questions on every ML project you ever do:

QuestionToolWhat to look for
1. Better than dumb luck?Compare with a baselineOur 79.8% vs a 50% coin flip. If a model can't clearly beat the dumbest strategy, it learned nothing.
2. Where does it fail?Confusion matrixBig numbers on the diagonal. Check which mistake it makes more — some mistakes cost more than others.
3. Can I trust a "yes"? Does it miss?Precision & recallPrecision = trustworthiness of its claims. Recall = how much it misses. Pick the one your problem cares about.
4. Was the exam fair?Train/test splitScore must be measured on data the model never saw. We kept 25% locked away for exactly this.

Rough accuracy guide for a balanced 50/50 problem like ours: ~50% = learned nothing · 60–75% = learned something · 75–90% = solid · 95%+ = be suspicious — usually the answer leaked into the clues.

The recipe you just learned

Every classification project follows this same skeleton — worth memorising:

  1. Load the data and look at it (Steps 2–3)
  2. Fix problems — we balanced 92/8 down to 50/50 (Step 4)
  3. Choose clues X and answer y (Step 5)
  4. Split 75% train / 25% test — and never let them mix (Step 6)
  5. Scale the features (Step 7)
  6. Train the model (Step 8)
  7. Judge honestly — baseline, confusion matrix, precision & recall (Steps 9–10)
  8. Tune the dials, like K (Step 11)

Swap KNeighborsClassifier for any other model and steps 1–7 stay exactly the same. That's why this recipe matters more than KNN itself.

Homework (optional, fun)

Because value = quantity × price, so the model can rebuild the missing clue from the other two. Now try keeping only "trade_value_crore": one clue alone still scores ~71.3%.   • Try to predict the impossible: change Step 5 to y = data["side"] (BUY vs SELL) and re-run. Accuracy crashes to ~50% — a coin flip. If the clues contain no signal about the answer, no model can save you. Knowing when ML won't work is as valuable as knowing when it will.
