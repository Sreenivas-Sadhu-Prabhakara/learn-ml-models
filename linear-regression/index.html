IDEA 1

The model is one weighted formula

After missing values, categories, and numeric scales are prepared, the model calculates:

The complete rule

prediction = intercept + weight₁ × feature₁ + weight₂ × feature₂ + … + weightₚ × featureₚ

The intercept is the starting point. A positive weight raises the prediction as that prepared feature rises; a negative weight lowers it. With several features, the formula describes a flat surface rather than a single line, but the rule is still “multiply, then add.”

Strength

The fitted rule is fast, inspectable, and often a strong benchmark for a numeric target.

Boundary

It cannot discover curves, thresholds, or feature interactions unless your chosen features express them.

Deep dive: what “best-fitting” means

Plain Linear Regression chooses weights that minimise the sum of squared training residuals. Squaring makes large errors count disproportionately. The course still selects and reports models with MAE because MAE is easier to read in the target’s units and is already the foundation’s primary regression score.

AUDIT BEFORE FIT

Remove shortcuts that will not exist in reality

Linear models make leakage especially visible because a leaked clue can receive a dominant weight—but by then the invalid result has already cost you time. Review each feature before fitting.

Leakage question Generic warning sign
Was this feature known at prediction time? A status, measurement, or total recorded after the outcome
Was it derived from the target? A ratio, band, flag, or duplicate calculated using the answer
Does it summarise other rows safely? An aggregate that includes the current row or future rows
Can the split reveal the same entity? Repeated people, devices, cases, or dates on both sides
Is preprocessing learned only from training folds? Imputation, scaling, or feature selection run before splitting

A near-perfect result is an audit trigger

It may be real, but first look for target-derived features, post-outcome fields, duplicate rows, and an unrealistic split. The pipeline below protects preprocessing boundaries; only your understanding of the problem can protect prediction-time boundaries.

IDEA 2

Plain, Ridge, or Lasso?

All three use the same weighted formula. Ridge and Lasso add a cost for large weights, called regularisation. That can make the formula less sensitive to noise and overlapping clues.

Family What fitting rewards Useful tendency
LinearRegression Small squared training errors Unpenalised reference model
Ridge Small errors plus a squared-weight penalty Shrinks correlated or unstable weights
Lasso Small errors plus an absolute-weight penalty Can set some weights exactly to zero

alpha controls penalty strength. Zero would remove the penalty; a very large value can flatten the model toward its intercept. The right family and alpha depend on your training data, so neither is chosen by hand here.

CELL 2 OF 5

Select the family and alpha inside training folds

The same pipeline prepares every candidate. Numeric scaling is important because Ridge and Lasso penalise coefficients: without comparable scales, a feature’s unit would change how expensive its weight appears.

CELL 2 · PIPELINE + MODEL SEARCH 
from sklearn.linear_model import LinearRegression, Ridge, Lasso
from sklearn.model_selection import GridSearchCV

linear_pipeline = Pipeline([
    ("prepare", make_preprocessor(scale_numeric=True)),
    ("model", LinearRegression()),
])

# Ridge is largely scale-free in y; Lasso's useful alpha range
# changes with the target's units, so adapt it to this dataset.
ridge_alphas = np.logspace(-3, 3, 7)
target_scale = max(float(np.std(y_train)), np.finfo(float).eps)
lasso_alphas = target_scale * np.logspace(-4, 0, 5)

search_space = [
    {
        "model": [LinearRegression()],
    },
    {
        "model": [Ridge()],
        "model__alpha": ridge_alphas,
    },
    {
        "model": [Lasso(max_iter=20000)],
        "model__alpha": lasso_alphas,
    },
]

linear_search = GridSearchCV(
    estimator=linear_pipeline,
    param_grid=search_space,
    scoring=PRIMARY_SCORING,
    cv=make_cv(),
    n_jobs=-1,
    return_train_score=True,
    error_score="raise",
)
linear_search.fit(X_train, y_train, **CV_FIT_PARAMS)

raw_cv = pd.DataFrame(linear_search.cv_results_)
cv_summary = pd.DataFrame({
    "family": [
        params["model"].__class__.__name__
        for params in raw_cv["params"]
    ],
    "alpha": [
        params.get("model__alpha", np.nan)
        for params in raw_cv["params"]
    ],
    "mean_validation_MAE": -raw_cv["mean_test_score"],
    "fold_to_fold_variation": raw_cv["std_test_score"],
    "rank": raw_cv["rank_test_score"],
})

display(
    cv_summary
    .sort_values(["rank", "family", "alpha"])
    .reset_index(drop=True)
)

best_linear_pipeline = linear_search.best_estimator_
selected_model = best_linear_pipeline.named_steps["model"]
selected_family = selected_model.__class__.__name__
selected_alpha = getattr(selected_model, "alpha", "not used")

print("Selected family:", selected_family)
print("Selected alpha: ", selected_alpha)
CHECKPOINT

You should see plain Linear Regression once, several Ridge and Lasso candidates, their validation MAE, and a selected model. Lower MAE is better. Close scores or large fold variation mean the ranking is uncertain.

Why this search is honest

Family, alpha, imputation, encoding, and scaling were all learned inside training folds. CV_FIT_PARAMS also preserves the foundation’s group boundary when your split mode uses groups. No test target influenced the choice.

Deep dive: a warning may still be useful evidence

Lasso can report that optimisation did not converge, especially with highly correlated features or an alpha that is too weak. Do not hide the warning. Increase max_iter, review redundant columns, or narrow the candidate range using training folds. Never choose a fix by looking at test performance.

CELL 3 · COEFFICIENT INSPECTION 
prepared_features = (
    best_linear_pipeline
    .named_steps["prepare"]
    .get_feature_names_out()
)
coefficients = np.asarray(selected_model.coef_).reshape(-1)
intercept = float(
    np.asarray(selected_model.intercept_).reshape(-1)[0]
)

coefficient_table = pd.DataFrame({
    "prepared_feature": prepared_features,
    "weight": coefficients,
})
coefficient_table["absolute_weight"] = (
    coefficient_table["weight"].abs()
)

print(f"Intercept: {intercept:.6g}")
print("Selected family:", selected_family)
print(
    "Weights near zero:",
    int(np.isclose(coefficients, 0).sum()),
    "of",
    len(coefficients),
)
display(
    coefficient_table
    .sort_values("absolute_weight", ascending=False)
    .head(20)
)
CHECKPOINT

You should see an intercept and up to 20 influential prepared columns. Numeric columns were standardised, while encoded categories are indicator columns, so their weights do not all represent the same kind of one-unit change.

Weights describe this fitted prediction rule—not causes

A large weight can reflect association, scale, category coding, correlation, or leakage. Correlated features may exchange weight while predictions remain similar. A zero Lasso weight means “unused by this fitted model under this alpha,” not “irrelevant in the world.”

CELL 5 OF 5

Inspect residuals, not only averages

A residual is actual - predicted. Positive means the model predicted too low; negative means it predicted too high. Reuse the final predictions to find systematic misses.

CELL 5 · RESIDUAL REVIEW 
outcomes = pd.DataFrame(
    {
        "__actual__": np.asarray(y_test),
        "__predicted__": np.asarray(linear_predictions),
    },
    index=X_test.index,
)
residual_review = pd.concat([outcomes, X_test], axis=1)
residual_review["__residual__"] = (
    residual_review["__actual__"]
    - residual_review["__predicted__"]
)
residual_review["__absolute_error__"] = (
    residual_review["__residual__"].abs()
)

display(
    residual_review
    .sort_values("__absolute_error__", ascending=False)
    .head(10)
)

low = min(
    residual_review["__actual__"].min(),
    residual_review["__predicted__"].min(),
)
high = max(
    residual_review["__actual__"].max(),
    residual_review["__predicted__"].max(),
)

fig, axes = plt.subplots(1, 3, figsize=(16, 4))

axes[0].scatter(
    residual_review["__actual__"],
    residual_review["__predicted__"],
    alpha=0.5,
)
axes[0].plot([low, high], [low, high], "--", color="black")
axes[0].set_xlabel("Actual")
axes[0].set_ylabel("Predicted")
axes[0].set_title("Actual vs predicted")

axes[1].scatter(
    residual_review["__predicted__"],
    residual_review["__residual__"],
    alpha=0.5,
)
axes[1].axhline(0, linestyle="--", color="black")
axes[1].set_xlabel("Predicted")
axes[1].set_ylabel("Residual")
axes[1].set_title("Residual pattern")

axes[2].hist(
    residual_review["__residual__"],
    bins=25,
    edgecolor="white",
)
axes[2].axvline(0, linestyle="--", color="black")
axes[2].set_xlabel("Residual")
axes[2].set_title("Residual distribution")

plt.tight_layout()
plt.show()
CHECKPOINT

Look for curvature, a widening funnel, separated clusters, consistently positive or negative regions, and a few dominant errors. Those patterns are more informative than whether the dots look “good.”

Pattern Possible interpretation Training-only next question
Curved residual band The weighted straight rule misses non-linearity Is a justified transformation or non-linear model appropriate?
Funnel shape Error spread changes with prediction size Would a target transformation or segment-specific analysis help?
Separate clusters A group may follow a different relationship Is a meaningful group feature missing or encoded poorly?
A few huge misses Outliers, data quality issues, or rare regimes Are those rows valid, important, and represented in training?

Do not repair the model against this test plot

The plot diagnoses the final result. If it motivates another modelling round, evaluate that round with training-only validation and obtain a fresh untouched holdout before making a new final claim.

RECAP

What you can now explain

  1. A linear prediction is an intercept plus weighted prepared features.
  2. Plain Linear Regression leaves weights unpenalised; Ridge shrinks them; Lasso can set some to zero.
  3. Scaling belongs inside the pipeline, especially when coefficient penalties are compared.
  4. Family and alpha are selected by cross-validation on training rows.
  5. Coefficients are model associations, not causal effects.
  6. Residual patterns expose errors hidden by MAE, RMSE, and R².
  7. Leakage is a problem-definition failure that no modelling technique can rescue.
Optional deep dive: feature engineering without leaking

A linear model can use a justified square, logarithm, rate, or interaction if it is computed from information available at prediction time. Add transformations inside a pipeline, describe why they match the domain, and compare them using training-only validation. Do not manufacture features after studying final test errors and then report the same test as untouched.
