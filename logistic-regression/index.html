What this model can answer

Use logistic regression when the target is a category: yes/no, one of several types, or another finite label. If your target is a measured number, return to the model map and choose a regression lesson.

THE IDEA

A boundary with a gradual transition

The model gives each row a linear score made from its transformed features and learned coefficients. A sigmoid turns a binary score into a probability estimate; for multiple classes, a softmax turns the class scores into estimates that sum to one. The class with the largest value becomes the default prediction.

Regularisation controls how freely coefficients can grow. A smaller C means stronger regularisation and a simpler boundary; a larger C gives the model more freedom. Neither direction is automatically better, so you will compare them inside the training set.

Probabilities need honest language

A value such as 0.80 is an estimated probability produced by this fitted model. It is not “80% certainty,” and it may be poorly calibrated when the sample is small, the future differs from training, or important information is missing.

CELL 5 OF 5

Read coefficients without overclaiming

The preprocessor may expand one original category into several indicator columns. The code below uses those transformed names and handles binary and multiclass targets separately.

CELL 5 · COEFFICIENTS 
prepared_feature_names = (
    best_logistic.named_steps["prepare"].get_feature_names_out()
)
fitted_logistic = best_logistic.named_steps["model"]

if len(fitted_logistic.classes_) == 2:
    coefficient_table = pd.DataFrame({
        "transformed_feature": prepared_feature_names,
        "coefficient": fitted_logistic.coef_[0],
    })
    coefficient_table["absolute_size"] = (
        coefficient_table["coefficient"].abs()
    )
    coefficient_table = coefficient_table.sort_values(
        "absolute_size", ascending=False
    )
    print(
        "Positive coefficients shift predictions toward:",
        fitted_logistic.classes_[1],
    )
    display(coefficient_table.head(20))
else:
    coefficient_table = pd.DataFrame(
        fitted_logistic.coef_.T,
        index=prepared_feature_names,
        columns=[str(label) for label in fitted_logistic.classes_],
    ).rename_axis("transformed_feature").reset_index()

    coefficient_long = coefficient_table.melt(
        id_vars="transformed_feature",
        var_name="class",
        value_name="coefficient",
    )
    coefficient_long["absolute_size"] = (
        coefficient_long["coefficient"].abs()
    )
    top_by_class = (
        coefficient_long
        .sort_values(["class", "absolute_size"], ascending=[True, False])
        .groupby("class", group_keys=False)
        .head(10)
    )
    display(top_by_class)
CHECKPOINT

For a binary target, signs refer to the second class shown. For a multiclass target, read each class column separately. Numeric features were scaled; categorical indicators were not, so coefficient sizes are not a universal importance ranking.

Coefficient does not mean cause

A coefficient describes the fitted relationship while the other transformed features are held fixed. Correlation among features, omitted variables, encoding choices, and regularisation can all change its sign or size.
