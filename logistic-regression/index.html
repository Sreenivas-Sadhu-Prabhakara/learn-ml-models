Lesson 03 · Classification

The model that tells you how sure it is

In Lesson 01, KNN gave you a verdict — BULK or BLOCK — and nothing else. Today's model, logistic regression, gives you the verdict plus a confidence score: "92% sure" vs "barely 51% sure". Same 1,19,156 real deals, same exam, one big new superpower.

⏱ about 25 minutes 🧠 builds on Lesson 01 (KNN) 📊 same data, same exam — scores comparable
Start the lesson ↓

The task, in two sentences

Same as Lesson 01: look at three numbers about a large Indian stock-market trade — quantity, price per share, total value in crore — and guess whether it was a BULK deal (big buying/selling spread across a normal trading day) or a BLOCK deal (one giant pre-negotiated trade in a special window). Last time the 5 nearest neighbours voted; this time the model draws the best dividing line through the data — and tells you how confident it is about every single call.

The whole idea, in one picture

Think of a land border between two states. Deep inside one state, there's no doubt where you are. Standing right on the border line? Could be either.

Logistic regression learns one straight dividing line between BULK-land and BLOCK-land. Then, for any new deal, it does two things: checks which side of the line the deal falls on (that's the verdict) and how far from the line it sits (that's the confidence). Far from the line → 95%+ sure. On the line → an honest 50/50 shrug.

One dividing line. Verdict = which side you're on. Confidence = how far from the line you sit.

It's the difference between a friend who shouts "it will rain!" and the weather app that says "90% chance of rain". Same forecast, but the second one lets you decide whether the umbrella is worth carrying.

STEP 3 OF 11

Balance the dataset to 50/50

Quick reminder of why: about 92% of the raw deals are BULK, so a lazy model shouting "BULK!" every time would score 92% while learning nothing. Exactly as in Lesson 01, we keep all 8,938 BLOCK deals and randomly pick an equal number of BULK deals — now blind guessing scores only 50%.

CELL 3 · PYTHON
# Step 3: Balance the dataset — the same 50/50 recipe as Lesson 01
block_deals = deals[deals["deal_type"] == "BLOCK"]
bulk_deals  = deals[deals["deal_type"] == "BULK"].sample(n=len(block_deals), random_state=42)

data = pd.concat([block_deals, bulk_deals])
data["deal_type"].value_counts()

STEP 4 OF 11

Same clues, same answer, same split

Clues (X): the three numbers. Answer (y): BULK or BLOCK. Then the honesty rule: 75% for studying, 25% locked away as the final exam — the model never sees the exam while learning.

CELL 4 · PYTHON
# Step 4: Same clues, same answer, same 75/25 split as Lesson 01
features = ["quantity", "price_inr", "trade_value_crore"]

X = data[features]        # the clues
y = data["deal_type"]     # the answer

X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    train_size=0.75,     # 75% training, 25% testing
    random_state=42,     # same split for everyone
    stratify=y,          # keep the 50/50 mix in both halves
)

print("Deals for training:", len(X_train))
print("Deals for testing: ", len(X_test))
WHAT YOU SHOULD SEE

Identical numbers to Lesson 01 — same 13,407 study deals, same 4,469 exam deals. Today's model sits the exact same exam KNN sat.

STEP 5 OF 11

Put all clues on the same scale

KNN needed scaling because it measures distances. Logistic regression wants it too, for its own reasons: it assigns each clue a weight (how hard that clue pushes the verdict), and weights are only comparable when the clues speak at the same volume. Scaling also helps the line-fitting finish quickly and cleanly. Same subtle detail as before: learn the scale from training data only, then apply it to the test data.

CELL 5 · PYTHON
# Step 5: Scale the clues so no single one dominates
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)   # learn the scale from training data
X_test_scaled  = scaler.transform(X_test)        # apply that same scale to test data

print("All clues are now on the same scale.")
WHAT YOU SHOULD SEE
All clues are now on the same scale.

Keep this cell in mind for Step 10 — the weights the model learns apply to these scaled clues, which is exactly what makes them fair to compare.

STEP 6 OF 11

Train the model

Unlike KNN — which just memorises the training deals — logistic regression genuinely learns here: it searches for the one dividing line that best separates the 13,407 training deals. On this data, that search still takes well under a second.

CELL 6 · PYTHON
# Step 6: Train the logistic regression model
logreg = LogisticRegression(random_state=42)
logreg.fit(X_train_scaled, y_train)

print("Model trained on", len(X_train), "deals.")
WHAT YOU SHOULD SEE
Model trained on 13407 deals.

About that terrible name. Despite being called logistic regression, this is a classifier — it sorts things into categories, exactly like KNN did. (In ML, "regression" normally means predicting a number, like a price.) The name is a historical hand-me-down from the 1800s: under the hood it borrows regression's line-drawing machinery, then squashes the line's output into a probability between 0% and 100%. Blame history, not yourself — everyone finds the name confusing at first.

STEP 7 OF 11

Exam time!

Predict all 4,469 locked-away test deals and check the answers against the truth.

CELL 7 · PYTHON
# Step 7: Exam time — predict the hidden 25%
predictions = logreg.predict(X_test_scaled)

accuracy = accuracy_score(y_test, predictions)
print(f"Accuracy: {accuracy:.1%}")
WHAT YOU SHOULD SEE
Accuracy: 75.3%

Is 75.3% good? Three reference points, all on this exact same exam:

  • Coin flip: ~50%. We clearly beat dumb luck — the model got about 3 out of 4 deals right.
  • Lesson 01's KNN: 79.8%. The single straight line loses about 4½ points to the neighbours' vote. A straight border is a blunt instrument, and this data's border is a bit curvy.
  • But wait for Steps 9 and 10. In exchange for those points, logistic regression gives you two things KNN never did: a smooth confidence score on every call, and an explanation of which clue pushed the decision. In many real jobs — medicine, lending — those are worth far more than 4 points of accuracy.

Judge the model properly

…plus the same numbers as a green-shaded chart. Reading it: of 2,235 real BLOCK deals, the model caught 1,425 and mislabelled 810. Of 2,234 real BULK deals, it caught 1,941 and mislabelled only 293.

Spot the personality. This model is a cautious inspector: it rarely cries "BLOCK!" without reason (only 293 false alarms), but it lets a lot of real BLOCK deals slip past (810 missed). KNN was more even-handed in both directions. Neither style is "better" — it depends on which mistake costs you more.

STEP 9 OF 11

The headline act: ask the model how sure it is

Everything so far, KNN could also do. Here's the new superpower. predict_proba ("predict probability") returns, for every deal, two percentages: the chance it's BLOCK and the chance it's BULK — always adding up to 100%. Let's attach the model's confidence to some real test deals.

CELL 10 · PYTHON
# Step 9: The headline act — ask the model HOW SURE it is
probabilities = logreg.predict_proba(X_test_scaled)
print("Column order:", logreg.classes_)

results = X_test.copy()
results["truth"]      = y_test.values
results["model_says"] = predictions
results["sure_%"]     = (probabilities.max(axis=1) * 100).round(1)

9c · Is the confidence honest? Check it.

A confidence score is only useful if it means something. So let's audit it: when the model claims to be very sure, is it right more often?

CELL 12 · PYTHON
# Step 9c: Is the confidence honest? Let's check.
confident = results[results["sure_%"] >= 90]
hesitant  = results[results["sure_%"] <  60]

right_confident = (confident["truth"] == confident["model_says"]).mean()
right_hesitant  = (hesitant["truth"]  == hesitant["model_says"]).mean()

print(f"When 90%+ sure ({len(confident)} deals): right {right_confident:.1%} of the time")
print(f"When under 60% sure ({len(hesitant)} deals): right {right_hesitant:.1%} of the time")
WHAT YOU SHOULD SEE
When 90%+ sure (633 deals): right 92.4% of the time
When under 60% sure (781 deals): right 51.5% of the time

The confidence is genuinely informative: on deals where it claims 90%+ certainty it scores 92.4%, and on deals where it admits it's nearly guessing, it performs like a coin flip — just as it warned you.

Why confidence is the superpower. Imagine two doctors reading the same scan. One says "it's fine." The other says "92% sure it's fine — but this one's a 55%, let's order one more test." Same for a bank: the loan model saying APPROVE at 99% confidence and APPROVE at 51% confidence are very different situations — the 51% file goes to a human officer. Confidence lets you automate the easy calls and escalate the doubtful ones. That's how ML is actually used in serious workplaces.

STEP 10 OF 11

Open the hood: which clue decides the verdict?

Second superpower: the model can show its working. Each clue gets a learned weight — how hard, and in which direction, that clue pushes the verdict. (This is why KNN felt like a black box: neighbours vote, but nobody tells you why.)

CELL 13 · PYTHON
# Step 10: Open the hood — which clue pushes which way?
weights = pd.DataFrame({
    "clue":   features,
    "weight": logreg.coef_[0].round(3),
})
print(weights.to_string(index=False))
print()
print("Positive weight → pushes the guess toward", logreg.classes_[1])
print("Negative weight → pushes the guess toward", logreg.classes_[0])
WHAT YOU SHOULD SEE
             clue  weight
         quantity  -0.003

Positive weight → pushes the guess toward BULK
Negative weight → pushes the guess toward BLOCK

In plain words — remembering these weights apply to the scaled clues from Step 5 (one "notch" = one typical-sized move in that clue, which is exactly what makes the three weights fair to compare):

ClueWeightWhat the model concluded
price_inr−6.786The star clue, by far. The pricier the share, the harder the push toward BLOCK. That's why the Step 9b deals — shares costing ₹5,000+ each — got near-100% BLOCK calls.
trade_value_crore−0.703A bigger total deal value gives a mild extra push toward BLOCK — the "block deals are bigger" pattern from Lesson 01.
quantity−0.003Almost zero. Once the model knows the price and the total value, the share count adds nearly nothing new.

Does the price story make sense? It does: block deals are institution-to-institution trades in big, established companies — whose shares tend to be pricier — while the bulk-deal list is full of low-priced small-company shares. Nobody told the model this. It found the pattern in the data, and now it can tell you it found it. Try explaining a decision like that with KNN.

The fun part: invent a deal, get a verdict with confidence

WHAT YOU SHOULD SEE
The model's verdict: BLOCK (76.1% sure)

Both models agree it's a BLOCK — and now you know it's a fairly confident BLOCK, not a certain one. Try the tiny deal from Lesson 01 — quantity 100000, price 50, value 0.5 — and it flips to BULK (74.5% sure). Then try an expensive stock — quantity 30000, price 5500, value 16.5 — and watch the confidence rocket to a rounded 100.0%.

How to judge any model — the cheat sheet

The four questions from Lesson 01, plus a fifth that exists only for models that report confidence:

QuestionToolWhat to look for
1. Better than dumb luck?Compare with a baselineOur 75.3% vs a 50% coin flip. If a model can't clearly beat the dumbest strategy, it learned nothing.
2. Where does it fail?Confusion matrixBig numbers on the diagonal — and check which mistake it favours. Ours misses BLOCK deals (810) far more than it falsely accuses (293).
3. Can I trust a "yes"? Does it miss?Precision & recallPrecision = when it claims BLOCK, how often is it right (0.83 here). Recall = how many real BLOCKs it catches (0.64 here). Pick the one your problem cares about.
4. Was the exam fair?Train/test splitScore must come from data the model never saw. Our 4,469 test deals stayed locked away until Step 7.
5. Is the confidence honest?predict_proba auditGroup predictions by claimed confidence and check the hit rate in each group — like our Step 9c (90%+ sure → 92.4% right; under 60% → coin flip).

The scoreboard so far, on the identical balanced data, identical split, identical 4,469-deal exam:

StrategyAccuracyTells you how sure?Explains itself?
Coin flip~50%NoNo
Logistic regression (this lesson)75.3%Yes — smooth 50–100% scoreYes — one weight per clue

The recipe you just reused

Notice what happened today: steps 1–5 and 7–8 were Lesson 01 verbatim. The only genuinely new lines were one import and one model. That's the whole point of the recipe:

  1. Load the data (Step 2)
  2. Fix problems — balance 92/8 down to 50/50 (Step 3)
  3. Choose clues X and answer y, split 75/25 (Step 4)
  4. Scale the clues (Step 5)
  5. Train — the only swapped ingredient (Step 6)
  6. Judge honestly — baseline, confusion matrix, precision & recall (Steps 7–8)
  7. NEW — ask how sure it is, and audit that confidence (Step 9)
  8. NEW — ask it to explain itself with one weight per clue (Step 10)

And the honest trade-off of the day: the straight line scored 75.3% against KNN's 79.8%, but it hands you a confidence score and an explanation with every prediction. In real deployments, that trade is taken all the time.
