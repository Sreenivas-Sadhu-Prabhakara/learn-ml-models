Learn a sequence of questions—and know when to stop

A decision tree repeatedly splits rows with simple rules. That makes a small tree readable, but an unrestricted tree can memorise training details. You will control that growth with training-only cross-validation.

THE IDEA

Every branch asks one narrow question

A split might ask whether a numeric feature is below a threshold or whether an encoded category is present. The algorithm chooses a question that makes the resulting groups more useful for predicting the target, then repeats inside each group.

For classification, leaves contain class distributions and produce class predictions. For regression, leaves predict a numeric value, usually the average target among training rows that reached that leaf.

Trees do not need scaled numbers

A threshold such as “feature ≤ value” keeps the same ordering if units are rescaled. This lesson still uses the shared preprocessor for missing values and categorical encoding, but it leaves numeric features in their original units.

Readable does not automatically mean trustworthy

A short branch can describe the fitted model clearly. It does not prove that the relationship is causal, fair, stable, or valid outside the training population.

CELL 2 OF 5

Control growth with cross-validation

max_depth limits how many questions a path can ask. min_samples_leaf prevents tiny leaves. The fractional option below means every leaf must contain at least that share of the training fold.

CELL 3 OF 5

See overfitting inside the training process

A flexible tree often scores extremely well on the folds it fits while falling behind on their validation rows. This train–validation gap is evidence of memorisation pressure; it is not something to diagnose by repeatedly checking the test set.

CELL 3 · TRAIN–VALIDATION GAP 
overfit_view = tree_cv_table.copy()
overfit_view["train_minus_validation"] = (
    overfit_view["mean_train_score"]
    - overfit_view["mean_test_score"]
)

display(
    overfit_view.sort_values(
        "train_minus_validation", ascending=False
    ).head(12)
)

best_cv_row = overfit_view.loc[
    overfit_view["rank_test_score"] == 1
].iloc[0]
print(
    "Chosen model train–validation gap:",
    best_cv_row["train_minus_validation"],
)
CHECKPOINT

For regression, the primary score is negative MAE, so larger is still better and the gap remains interpretable. A large gap signals instability; a small gap does not prove the model is useful.
