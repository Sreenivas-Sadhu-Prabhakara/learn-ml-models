Average many varied trees for a steadier model

THE IDEA

Many deliberately different trees

Each tree sees a bootstrap sample of the training rows and considers only a subset of features at each split. Those differences reduce the chance that every tree repeats the same error. Classification combines class estimates; regression averages numeric predictions.

More trees usually make the average more stable, while depth, leaf size, and the number of candidate features control how complex and similar the individual trees become.

No numeric scaling required

Like a single decision tree, a forest works through ordered thresholds. The shared preprocessor still fills missing values and encodes categories, but leaves numeric units unchanged.

Power does not remove the need for a baseline

A more complex model is worthwhile only if its validation behaviour, held-out performance, operational cost, and error pattern suit the real problem.

CELL 4 OF 6

Measure reliance on original columns

Permutation importance shuffles one input column at a time and measures the performance drop. Because it receives the complete pipeline and the original X_test table, one result corresponds to each column in FEATURES—not to each one-hot encoded column.

CELL 4 · PERMUTATION IMPORTANCE 
permutation_result = permutation_importance(
    best_forest,
    X_test,
    y_test,
    scoring=PRIMARY_SCORING,
    n_repeats=8,
    random_state=RANDOM_STATE,
    n_jobs=-1,
)

importance_table = pd.DataFrame({
    "original_feature": FEATURES,
    "mean_score_drop": permutation_result.importances_mean,
    "variation": permutation_result.importances_std,
}).sort_values("mean_score_drop", ascending=False)

display(importance_table)

plot_table = importance_table.head(20).sort_values("mean_score_drop")
plt.figure(figsize=(9, max(4, 0.35 * len(plot_table))))
plt.barh(
    plot_table["original_feature"],
    plot_table["mean_score_drop"],
    xerr=plot_table["variation"],
)
plt.xlabel(f"Drop in {PRIMARY_SCORING} after shuffling")
plt.title("Permutation importance on original input columns")
plt.tight_layout()
plt.show()
CHECKPOINT

A large positive drop means the fitted pipeline relied on that column for these held-out rows. Near-zero or negative values can occur through noise, weak features, or shared information between correlated columns.

Reliance is association, not causation

Shuffling asks how this model’s score changes when a column’s information is broken. It does not show that changing the real-world feature will change the outcome. Correlated columns can share or hide importance, and leaked features can appear highly important.

CELL 5 OF 6

Inspect probabilities only for classification

A classifier averages class estimates across its trees. These outputs can support threshold decisions, but they are still estimates and are not guaranteed to match real-world frequencies. Regression forests do not produce class probabilities.

CELL 5 · CONDITIONAL PROBABILITIES
